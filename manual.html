로그인하기

회사 이메일과 비밀번호로 GoWIT IDP 시스템에 접속합니다. 로그인 후 역할(role)에 따라 이동되는 화면이 다릅니다.

로그인 순서
1
login.html 접속
브라우저에서 시스템 URL에 접속합니다. 상단의 "시스템 접속" 버튼을 눌러도 됩니다.
2
이메일·비밀번호 입력
회사 이메일 주소와 비밀번호를 입력합니다.
예시
이메일: team1.kim@gowit.co.kr
비밀번호: 1234
3
로그인 버튼 클릭
입력 후 로그인 버튼을 클릭하면 역할에 따라 자동으로 이동됩니다.
로그인 후 이동 화면
역할로그인 후 이동설명
admin admin.html 관리자 콘솔 화면으로 즉시 이동
manager index.html 사용자 메인 화면 (IDP 합의·피드백 작성 가능)
user index.html 사용자 메인 화면 (IDP 작성·조회)
자동 이동이 되지 않나요? 이메일 또는 비밀번호가 맞지 않으면 오류 메시지가 표시됩니다. 비밀번호를 잊은 경우 관리자(admin@gowit.co.kr)에게 문의하세요.
인증 보안 방식

GoWIT IDP는 로그인 성공 시 sessionStorage에 세션 정보를 저장합니다. 브라우저 탭을 닫으면 세션이 만료되어 다시 로그인해야 합니다.

  • 브라우저 탭 닫기 → 세션 만료 → 재로그인 필요
  • 미인증 상태로 index.html/admin.html 직접 접속 → 자동으로 login.html 이동
  • 일반 계정으로 admin.html 접속 시도 → 자동 차단 및 login.html 이동

역량사전

직무역량 7개와 리더십역량(C1~C4 밴드별) 전체 역량의 정의·행동 기준·AI 활용 포인트를 조회합니다.

역량 구성
구분항목 수적용 대상레벨 체계
직무역량7개전 밴드 공통5단계 (이해→적용→분석→판단/평가→창의/혁신)

  • 레벨별 행동 기술서 (직무역량: 5단계 / 리더십: 3단계)
  • AI 활용 포인트 — 각 역량을 AI 도구로 강화하는 방법
  • 핵심 행동지표 목록
  • 개발 목표 및 권장 학습 가이드 (리더십역량)

역량수준 진단

본인의 현재 역량 수준을 역량별로 자가 진단합니다. 진단 결과는 IDP 작성 시 목표 레벨 설정의 기준이 됩니다.

진단 방법
1
역량별 점수 입력
각 역량을 1~5점으로 평가합니다. 1점(이해 수준) ~ 5점(창의/혁신 수준)
2
진단 제출
진단 완료 버튼을 클릭하면 결과가 저장됩니다.
3
레이더 차트 확인
제출 후 직무역량·리더십역량 레이더 차트로 결과가 시각화됩니다. 대시보드에도 반영됩니다.
IDP 작성과 연동됩니다 진단 결과(현재 레벨)는 IDP 작성 시 '현재 수준' 항목에 자동으로 불러와집니다. 진단 후 IDP를 작성하면 더욱 정확한 목표 설정이 가능합니다.

IDP 작성하기

개인 역량 개발 계획(IDP)을 4단계 위저드 방식으로 작성합니다. 중간에 임시저장이 가능하므로 한 번에 완성하지 않아도 됩니다.

역량수준 진단 먼저! IDP 작성 전 역량수준 진단을 먼저 완료하면 '현재 수준'이 자동으로 불러와져 작성이 훨씬 편리합니다.
작성 단계 (4단계)
1
역량 목표 선택
역량사전에서 개발할 역량을 선택하고 목표 레벨(현재 레벨 + 1 권장)을 설정합니다.
💡 추천 방법
역량수준 진단 결과를 바탕으로 가장 GAP이 큰 역량 1~2개를 선택하세요.
목표 레벨은 현재 레벨보다 1단계 높게 설정하는 것이 현실적입니다.
2
실행 계획 수립
역량별로 구체적인 실행 과제, 기간, 방법을 입력합니다.
입력 항목
· 실행 과제명: 무엇을 할 것인지 구체적으로
· 기간: 시작일 ~ 종료일
· 방법: 교육/OJT/자기학습/코칭 등
· 달성 기준: 어떻게 완료를 확인할지
3
합의 라인 확인
합의 라인은 밴드·직책에 따라 2단계로 자동 구성됩니다. 별도 지정 없이 제출하면 시스템이 적절한 합의자를 자동으로 설정합니다.
예시 합의 라인 (김팀원 C1의 경우)
1차: 한파트 (C3 파트장, 중간합의) → 2차: 유팀장 (C4 팀장, 최종합의)
4
검토 & 제출
전체 내용을 최종 확인한 후 제출 버튼을 클릭합니다. 제출과 동시에 1차 합의자에게 합의 요청이 전달됩니다.
임시저장 사용법

작성 중 임시저장 버튼을 클릭하면 현재까지 입력한 내용이 저장됩니다. 이후 "IDP 작성" 메뉴에 재진입하면 이어서 작성할 수 있습니다.

임시저장 주의사항 임시저장 상태에서는 합의자에게 요청이 전달되지 않습니다. 반드시 제출까지 완료해야 합의가 시작됩니다.

IDP 현황 확인

내가 작성한 IDP의 진행 상태를 확인하고, manager 역할은 담당 구성원의 IDP 현황도 조회할 수 있습니다.

IDP 상태 종류
임시저장

테스트 시 주의사항

GoWIT IDP 시스템은 현재 내부 테스트 버전(v1.3)입니다. 테스트 전 아래 사항을 반드시 확인하세요.

localStorage 저장
모든 데이터는 브라우저 localStorage에 저장됩니다. 브라우저·기기별 데이터가 분리됩니다.
캐시 삭제 주의
브라우저 캐시(사이트 데이터) 삭제 시 테스트 데이터가 모두 초기화됩니다.
이메일 인증 한도
EmailJS 무료 한도는 월 200건입니다. 초과 시 인증 메일 발송이 불가합니다.
다중 사용자 테스트
동일 PC·브라우저에서 계정 전환 시 로그아웃 후 재로그인이 필요합니다.
버그 발견 시
화면명, 재현 단계, 오류 메시지를 담당자(HR팀)에게 공유해 주세요.
외부 공유 금지
이 시스템과 가이드는 내부 테스트 목적으로만 사용되며, 외부 공유가 금지됩니다.
문의 및 피드백 테스트 완료 후 발견된 사항은 juyeon.kim@gowit.co.kr로 전달해 주시면 감사합니다.
